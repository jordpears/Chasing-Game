Breakout Write-Up

Aim: The aim of this project was to further my javascript skills being the second significant project I have undertaken, with the first one being the very rudamental mouse chasing game which has no mobile adaptions and is clunky. I wanted to do the project in pure JS and handle all mouse events manually.

Execution: I used a HTML5 canvas to host the game and made this canvas match the size of the users screen for a full-screen experience. I created objects for each block, ball, and powerup. I also used an object to represent the dimensions of the paddle. Drawing from my experience developing the mouse chasing game I focussed on making every element of the game dynamically sized ensuring that the game will be correctly sized for any device using it. At the beginning of the game the parameters for the dimensions of every object is determined from the users size then the screen is populated with a selection of blocks of random primary colors and the paddle is placed on the screen at a set y-offset. This y-offset is increased significantly if the user is on a mobile to ensure that the paddle can be seen under the finger. The user starts with a new ball object on their paddle, the ball objects are inactive and do not move until the user clicks/taps the screen, which will deploy them in a random direction. The game updates at 60fps. Every iteration the balls are moved based on their velocity variables and the paddle is updated every time the user moves their mouse. Every iteration it is checked if any of the ball locations overlap with a paddle or a block or the game boundaries. If so they bounce reversing one of the x or y velocities depending on the direction of collision. The balls are contained in an array and are removed from the array if their y position goes over the bottom of the screen. If the array is empty then the game is lost and all elements are reset for the player to try again. As a bonus for players who may find the max speed too fast, every time a player loses the max speed will slightly decrease, making the game more accessible. There are also randomly dropping powerups which are created everytime a block is broken, these can be yellow or red. Red increases the size of the paddle up to some max value, and yellow gives the user an additional ball ready to deploy, these balls can be saved and deployed en masse. When all blocks are removed from the array containing them (done upon destruction) the game is won and for some fun 50 balls are randomly generated and activated on the screen for the player to knock about until all balls fall off the screen and the game is "lost" and regenerates. When the ball hits the paddle its rebound direction is determined by the side of the paddle it hits as in the original breakout game.

How could it be improved: I chose to use a square collision box for the balls, this could be improved by using a circular one but would increase the calculations for each iteration and this rarely causes a problem. If i wanted more realistic collision mechanics I could have created a probe that checks if the next collision will pass through a block location because sometimes the distance of position change due to high velocity can clip through the corner of blocks. I could also add more gaming elements, it would not be challenging to add additional levels with varying block layouts or multiple hit blocks but I do not think I would learn much from this exercise.
